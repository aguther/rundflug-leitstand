# Web-Anwendung

- Die aktuelle Oberfläche ist ein technischer Startbildschirm, keine freigegebene Produktgestaltung.
- Vor fachlicher UI-Implementierung vollständige Konzepte für Kasse, Flight Line, FIDS und Ticketstatus erstellen und freigeben lassen.
- Kasse und Flight Line bleiben im Standardablauf Ein-Bildschirm-Oberflächen.
- Keine Fachregeln in React-Komponenten duplizieren.
- Touch-Ziele, Sonnenlichttauglichkeit, Kioskmodus und Offlinezustand explizit testen.
