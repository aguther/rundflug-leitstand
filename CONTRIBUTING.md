# Mitwirken

1. Issue oder Arbeitspaket mit Anforderungs-IDs anlegen.
2. Branch nach Muster `codex/F-KAS-010-ticketverkauf` erstellen.
3. Plan in `PLANS.md` oder im Pull Request führen.
4. Tests parallel zur Implementierung ergänzen.
5. `npm run check` ausführen.
6. Traceability aktualisieren.
7. Pull Request mit Risiken, Screenshots und ausgeführten Tests einreichen.

Commits sollen klein, nachvollziehbar und frei von generierten Binärdateien außerhalb der vorgesehenen
Dokumentation sein.
